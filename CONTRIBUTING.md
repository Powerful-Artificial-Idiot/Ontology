# Contributing

1. Create a focused branch and keep Demo interaction changes separate from ontology changes.
2. Run `make validate`, `make test`, and `make demo-build` before opening a pull request.
3. Add a migration record for breaking or renamed ontology terms.
4. Add or update SHACL, mappings, fixtures, and competency questions with every semantic change.
5. Do not commit source-system credentials, production data, or generated build outputs.
